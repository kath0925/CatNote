# Software Engineering Discipline and Practice

这门课结合线下讲座、实验和讨论，重点不是单纯学习某个编程语法，而是理解软件项目如何从想法走到交付。

课程会覆盖软件工程过程、团队项目、GitHub 协作、需求、设计、测试、用户体验评估、项目管理、可持续性和隐私等主题。

## Assessment

- 50% 项目与报告：团队在 GitHub 上完成一个带有 innovation twist 的游戏项目，并提交代码、报告、demo 和视频。
- 50% 笔试考试：以简答题为主，考察你能否解释概念、比较方法，并把软件工程知识应用到具体场景。

## Project

- 团队项目使用 P5.js 和 JavaScript，通常以 2D 游戏为主，更看重软件工程过程质量，而不是游戏点子本身。
- 课程强调识别软件开发挑战、团队协作、版本管理、过程记录和反思，而不是只交出一个能运行的小游戏。
